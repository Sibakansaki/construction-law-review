# 施工法複習網站 — 開發規格文件

## 1. 專案概述

- **專案名稱**：施工法複習網站
- **主要用途**：個人備考複習，透過知識點卡片與 AI 互動強化記憶
- **執行方式**：直接用瀏覽器開啟 HTML 檔（本地端，無需伺服器）
- **技術選擇**：純 HTML / CSS / JavaScript，單一資料夾結構
- **資料儲存**：localStorage（狀態、筆記、答題記錄）+ 本地 JSON 檔（知識點內容）

---

## 2. 資料夾結構

```
project/
├── index.html
├── style.css
├── app.js
├── api.js              ← API 串接邏輯
└── data/
    ├── ch2.json
    ├── ch3.json
    ├── ch4.json
    └── ...             ← 每章一個 JSON 檔
```

---

## 3. 知識點卡片 JSON 格式

每章一個 JSON 檔，格式如下：

```json
{
  "chapter": "CH2",
  "title": "土方工程",
  "cards": [
    {
      "id": "ch2-001",
      "name": "知識點名稱",
      "content": "知識點內容說明..."
    },
    {
      "id": "ch2-002",
      "name": "知識點名稱",
      "content": "知識點內容說明..."
    }
  ]
}
```

> JSON 檔由 Claude.ai 讀取課本 PDF 後生成，手動存入 /data 資料夾。

```

- 顯示該章節所有知識點卡片（全部列出，可上下滾動）
- 「開始本章測驗」按鈕：針對整章出申論題

### 4.3 知識點卡片

```
┌─────────────────────────────────────┐
│  知識點名稱                          │
│                                     │
│  內容說明文字...                     │
│                                     │
│  [ 熟練 ]  [ 待加強 ]  [ 筆記 ]  [ 測驗 ] │
└─────────────────────────────────────┘
```

**狀態按鈕行為：**
- 「熟練」：點後卡片變綠色邊框，狀態存入 localStorage
- 「待加強」：點後卡片變橘色邊框，狀態存入 localStorage
- 預設：無顏色（未標記）

### 4.4 筆記面板（點「筆記」後展開於卡片下方）

```
┌─────────────────────────────────────┐
│  📝 筆記                            │
│  ┌─────────────────────────────┐   │
│  │ 可輸入文字區域...            │   │
│  └─────────────────────────────┘   │
│  [ 儲存筆記 ]                       │
│                                     │
│  [ ✨ AI 幫我想口訣 ]               │
│  ┌─────────────────────────────┐   │
│  │ AI 生成的記憶技巧顯示於此   │   │
│  └─────────────────────────────┘   │
└─────────────────────────────────────┘

- 筆記內容存入 localStorage，key 為卡片 id

- AI 生成的口訣快取至 localStorage，避免重複呼叫

- AI 根據整章內容出一題申論題

- 提交後顯示評分、答對部分、遺漏部分、完整詳解

## 5. AI 功能規格

### 使用模型
- 預設模型：`claude-sonnet-4-5`（寫死在 api.js，不開放設定）

### API 呼叫格式

```javascript
const response = await fetch(`${endpoint}/v1/messages`, {
  method: "POST",
  headers: {
    "Content-Type": "application/json",
    "x-api-key": apiKey,
    "anthropic-version": "2023-06-01"
  },
  body: JSON.stringify({
    model: "claude-sonnet-4-5",
    max_tokens: 1000,
    messages: [{ role: "user", content: prompt }]
  })
});
```

### AI 功能對應的 Prompt

**① AI 幫我想口訣**
```
你是一位記憶技巧專家。以下是一個施工法的知識點：

知識點名稱：{name}
內容：{content}

請幫我設計一個好記的口訣或聯想記憶法，讓我能快速背起這個知識點的重點。
請用繁體中文回答，並說明這個口訣怎麼對應到實際內容。
```

**② 單一知識點出申論題**
```
你是一位施工法的考試出題老師。以下是一個知識點：

知識點名稱：{name}
內容：{content}

請根據此知識點出一題申論題，難度適合國家考試程度。
只需輸出題目本身，不要包含答案。
```

**③ 本章出申論題**
```
你是一位施工法的考試出題老師。以下是本章的知識點清單：

{章節所有知識點的名稱與內容}

請根據本章內容出一題涵蓋範圍較廣的申論題，難度適合國家考試程度。
只需輸出題目本身，不要包含答案。
```

**④ AI 批改申論題**
```
你是一位施工法的閱卷老師，請批改以下申論題的答案。

題目：{question}
知識點參考內容：{content}
學生答案：{studentAnswer}

請用以下格式回答：
評分：＿／100
✅ 答對的部分：（條列說明）
❌ 遺漏或錯誤的部分：（條列說明）
📖 完整詳解：（完整標準答案與說明）

請用繁體中文回答。
```

## 6. localStorage 資料結構

```javascript
// 卡片狀態
"card_status_{cardId}" : "熟練" | "待加強" | ""

// 卡片筆記
"card_note_{cardId}" : "筆記內容..."

// AI 口訣（快取，避免重複呼叫）
"card_mnemonic_{cardId}" : "AI 生成的口訣..."

// 答題記錄
"quiz_records" : [
  {
    date: "2025/04/21",
    cardId: "ch2-001",         // 本章測驗則為 "chapter-CH2"
    cardName: "知識點名稱",
    question: "題目...",
    studentAnswer: "我的答案...",
    score: "85／100",
    feedback: "AI 詳解..."
  }
]

// API 設定
"api_endpoint" : "https://cc.zhihuiapi.top"
"api_key" : "sk-..."
```

## 7. 視覺風格

- **配色**：深色系為主（深灰背景、白色文字）、簡潔專業
- **語言**：繁體中文
- **裝置**：以電腦為主（不特別做手機響應式）
- **卡片狀態顏色**：
  - 熟練：綠色邊框
  - 待加強：橘色邊框
  - 未標記：預設灰色邊框

---

## 8. 開發注意事項

- 直接開啟 HTML 檔（file:// 協定），JSON 資料改用 JS 模組格式（`export default {...}`）並以 `import` 讀取，避免 fetch 被瀏覽器封鎖
- 所有 AI 功能呼叫前先檢查 localStorage 是否有 API Key，若無則跳出提示引導至設定
- API Key 只存在 localStorage，不寫入任何程式碼檔案
