# 考古題題庫建立器

## 核心原則：以知識點卡片為主體

> **考古題是用來「標記卡片重要性」的工具，不是建立新知識的來源。**

- 對照方向是：**從卡片出發，去看考古題有沒有考到它**
- 不是：從考古題出發，去找卡片
- 考古題題目若與現有卡片（ch2～ch12）**完全無重疊**，直接忽略，不納入 examBank.js
- 時代已過、課程已不涵蓋的舊題目，跳過不處理

---

## 目標

1. 讀取使用者上傳的考古題 PDF
2. **以現有卡片（ch2～ch12）為基準**，篩選出有命中卡片的題目
3. 無重疊的題目直接略過，不輸出、不記錄
4. 輸出 `examBank.js`（僅含命中卡片的題目）
5. 輸出各章更新版 `.js`，補強 `examYears` 與 `isExamFocus` 欄位
6. 為申論題 AI 出題提供精準的歷史命題方向

---

## 知識點卡片架構

使用者的複習網站卡片涵蓋 CH2～CH12，每張卡片有唯一 `id`，格式為：
- `ch2-2.1.1`、`ch10-10.2.1a`、`ch11-11.1.2b` 等

**各章主題對照：**
| 章節 | 主題 |
|------|------|
| CH2 | 假設工程 |
| CH3 | 開挖及擋土工程 |
| CH4 | 基礎工程 |
| CH5 | 混凝土工程 |
| CH6 | 鋼結構工程 |
| CH7 | 模板工程 |
| CH8 | 鋼筋工程 |
| CH9 | 預力混凝土工程 |
| CH10 | 隧道及管道工程 |
| CH11 | 道路及山坡地工程 |
| CH12 | 生態及永續工程 |

## 讀取考古題 PDF 的原則

1. **完整閱讀全文**，記錄每一題的題型、題目內容、關鍵字
2. **識別題型：**
   - 申論題（問答題、計算題、說明題）
   - 選擇題（若有）
3. **識別考試資訊：**
   - 年份（如 114年、113年）
   - 考試別（高考三級、普考、土木技師、結構技師、司法特考、鐵路特考、鐵路員級等）
   - 科目（施工法、土木施工學等）
4. **每一題逐題對照知識點卡片（以卡片為主體）：**
   - 先判斷題目的核心知識點是否存在於現有卡片（ch2～ch12）中
   - **有命中 → 記錄對應的 card id，納入 examBank**
   - **無命中（題目涉及的知識點不在任何卡片中）→ 直接忽略，不納入、不記錄**
   - 一題可對應多張卡片（如題目跨兩個工法比較），全部列出

## 輸出一：examBank.js

```js
export default [
  {
    "id": "eq-001",
    "year": 114,
    "exam": "高考三級",         // 考試別
    "subject": "施工法",         // 科目名稱
    "type": "申論",              // 申論 or 選擇
    "question": "試述潛盾工法之施工原理，並說明環片組立及防水措施之要點。",
    "relatedCards": ["ch10-10.2.1a", "ch10-10.2.1b"],  // 對應卡片 id（一定有值）
    "keywords": ["潛盾工法", "環片", "防水"],
    "chapter": "CH10"
    // ※ 無命中卡片的題目不會出現在此檔案中
  },
  {
    "id": "eq-002",
    "year": 113,
    "exam": "土木技師",
    "subject": "施工法",
    "type": "申論",
    "question": "新奧工法（NATM）之施工流程為何？隧道淨空變位過大時應如何處理？",
    "relatedCards": ["ch10-10.1.2a", "ch10-10.1.2b"],
    "keywords": ["NATM", "新奧工法", "淨空變位"],
    "chapter": "CH10"
  }
]
```

### id 命名規則
- 格式：`eq-{三位數流水號}`，如 `eq-001`、`eq-002`
- 跨年度、跨考試別統一編號，不重置

## 輸出二：各章 .js 更新版（examYears 補強）

讀取現有各章 `.js` 後，依照 examBank 的對照結果，更新每張卡片的：

```js
"isExamFocus": true,           // 只要有任一考試考過，改為 true
"examYears": [                 // 精確到年份＋考試別
  "114年高考三級",
  "112年土木技師",
  "108年普考"
],
"examHint": "常考施工流程與環片防水三大措施"  // 從考古題方向歸納
```

### 從未被考古題命中的卡片
加入欄位 `"neverTested": true`，讓網站可用不同樣式標記（如黑框、灰底）：

```js
"isExamFocus": false,
"examYears": [],
"examHint": "",
"neverTested": true            // 新增：從未出現在任何考古題中
```

## 輸出三：申論題出題用 System Prompt 片段

每次處理完一批考古題後，同時輸出一份申論出題 prompt 建議片段供網站使用：

```
【歷年考古題命題方向（依章節）】

CH10 隧道及管道工程：
- 114高考：潛盾工法環片防水（ch10-10.2.1a、10.2.1b）
- 113土木技師：NATM施工流程與淨空變位處理（ch10-10.1.2b）
- 112普考：管幕工法施工步驟（ch10-10.2.3）

CH11 道路及山坡地工程：
- 113高考：瀝青混凝土滾壓三階段（ch11-11.1.2b）
- 111土木技師：加勁擋土牆施工方法（ch11-11.2.2b）
...

【出題規則】
- 70% 優先從以上歷年考古題的方向與角度命題
- 30% 從你認為重要、尚未被考過的知識點（neverTested: true 的卡片）命題
- 申論題字數：每題建議答案 200~400 字
- 題型格式模擬真實考試（「試說明…」「試比較…」「試述…並說明…」）
```

## 作業流程

### Step 1：使用者上傳考古題 PDF
確認以下資訊：
- 幾年份？哪些考試別？
- 是否已有舊版 examBank.js 需要合併？

### Step 2：讀取 PDF，建立題目清單
逐題整理：年份、考試別、題型、題目文字、關鍵字

### Step 3：逐題對照卡片（以卡片為主體篩選）
- 先判斷題目核心知識點是否在現有卡片範圍內
- **有命中 →** 記錄對應 card id，納入 examBank
- **無命中 →** 直接略過，不記錄、不輸出
- 一題多卡片時全部列出

### Step 4：輸出檔案
| 檔案 | 說明 |
|------|------|
| `examBank.js` | 完整考古題題庫（累積所有年份） |
| `ch{n}.js`（更新版） | 已補強 examYears 的各章卡片 |
| `examPrompt.txt` | 申論出題用 prompt 片段（純文字） |

所有檔案存入 `/mnt/user-data/outputs/` 並用 `present_files` 提供下載。

### Step 5：告知使用者
完成後回報：
1. 本批讀取了幾道題目，其中幾道命中卡片、幾道略過（無對應卡片）
2. 命中了哪些章節的哪些卡片（列出 top 5 最常考的卡片）
3. 哪些卡片從未被任何考古題命中（neverTested）
4. 是否需要繼續上傳下一批

---

## 注意事項

- **以卡片為主體，考古題為輔：** 卡片沒有的知識點，考古題考到也忽略
- **無命中的題目完全略過：** 不記錄、不輸出、不提示「此題超出範圍」
- **同一題可對應多張卡片**，全部列入 `relatedCards`
- **考試別統一寫法：** 高考三級、普考、土木技師、結構技師、大地技師、司法特考、鐵路特考、鐵路員級、地方特考
- **examBank.js 為累積檔案**：每次新增年份時，合併至現有 examBank.js，不覆蓋舊資料
- **id 流水號不重置**：新增題目接續最後一個 id 編號往下排
