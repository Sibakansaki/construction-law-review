# PDF 課本 → 知識點卡片 JS 檔

## 目標
將施工法課本 PDF 章節整理成供複習網站使用的 `.js` 知識點卡片資料檔。

---

## 閱讀 PDF 的原則

1. **完整閱讀全文**，不跳過任何頁面
2. **識別章節結構**：
   - `x.x`（如 3.1）→ section 分割線
   - `x.x.x`（如 3.1.1）→ 一張知識點卡片
   - 若一個 x.x.x 內容過長（超過5個重點），可拆成多張卡片（在 name 後加 `—` 副標題）
3. **識別課本內嵌的考古題**：
   - 通常格式為【基本實務題】或【進階應用題】方框，附有年份（如 103年土木技師）
   - 若考古題對應到某知識點，標記 `isExamFocus: true`，填入 `examYears` 和 `examHint`

---

## 卡片內容撰寫規則

### 精簡原則
- 每張卡片抓 **3~5 個核心重點**，不把課本整段搬過來
- 優先保留：數字、法條來源、定義、優缺點、施工步驟、注意事項
- 刪除：過多背景說明、重複敘述、過於細節的補充

### HTML 格式規範
使用 HTML 標籤讓卡片有條理：

```html
<!-- 粗體重點 -->
<strong>重要名詞或數字</strong>

<!-- 條列（無序） -->
<ul><li>項目一</li><li>項目二</li></ul>

<!-- 條列（有序步驟） -->
<ol><li>步驟一</li><li>步驟二</li></ol>

<!-- 比較表格（表頭一律用淡藍色，不可用黑色） -->
<table style='width:100%;border-collapse:collapse;font-size:0.9em'>
  <tr style='background:#dbeafe'>
    <th style='border:1px solid #93c5fd;padding:6px;color:#1e40af;font-weight:700'>欄位</th>
  </tr>
  <tr>
    <td style='border:1px solid #93c5fd;padding:6px'>內容</td>
  </tr>
</table>

<!-- 換行 -->
<br>
```

### 什麼時候用表格
- 多個工法的優缺點比較
- 多個人員職責對照
- 多個法規條文對應表

## 輸出格式

```js
export default {
  "chapter": "CH3",                        // 章號，如 CH2、CH3
  "title": "開挖及擋土工程",               // 章節標題
  "sections": [
    {
      "id": "3.1",                          // section 編號
      "title": "3.1 開挖工法",             // section 標題（含編號）
      "cards": [
        {
          "id": "ch3-3.1.1",               // 唯一 ID，格式：ch{章號}-{小節編號}
          "name": "3.1.1 斜坡明挖工法",   // 卡片標題（含編號）
          "content": "<strong>定義：</strong>...",  // HTML 格式內容
          "isExamFocus": false,            // 是否為考點
          "examYears": [],                 // 出現的考試年份，如 ["103年土木技師"]
          "examHint": "",                  // 考試方向提示，如 "常考施工步驟與注意事項"
          "quiz": [                        // 每張卡片固定 6 題多選題
            {
              "question": "題目文字",
              "options": ["A. 選項一", "B. 選項二", "C. 選項三", "D. 選項四"],
              "answer": ["A", "C"],        // 正確答案（可1~多個字母）
              "explanation": "解析文字"
            }
            // ... 共 6 題
          ]
        }
      ]
    }
  ]
}
```

### ID 命名規則
- section id：直接用編號，如 `"3.1"`、`"3.2"`
- card id：`ch{章號}-{完整小節編號}`，如 `"ch3-3.1.1"`
- 若一個小節拆成多張卡片：`"ch3-3.2.5a"`、`"ch3-3.2.5b"`

---

## Quiz 出題規範

每張卡片必須包含 **6 題多選題**（`quiz` 陣列），規格如下：

### 題目格式
- 每題固定 **4 個選項**（A B C D）
- 答案可為 **1~4 個**，存入 `answer` 陣列（如 `["A", "C"]`）
- **不標示「單選」或「多選」**，讓學生自行判斷
- 附上 `explanation` 解析，說明每個選項正確或錯誤的理由

### 出題品質原則
1. **考點分散**：6 題盡量覆蓋卡片內不同知識點，不重複考同一細節
2. **錯誤選項有意義**：錯誤選項應為常見混淆概念，不可太離譜或無關
3. **全錯題型**：至少 1 題設計為「4 個選項全部正確」（answer 為 `["A","B","C","D"]`），另設計 1 題為「4 個選項全部錯誤」（answer 同樣列出所有應選但題目問「何者有誤」）
4. **全選答案題**：部分題目可設計讓 3~4 個選項均正確，訓練全面掌握
5. **解析清楚**：解析需說明所有選項正確或錯誤的理由，特別指出易混淆點

### Quiz 範例
```js
"quiz": [
  {
    "question": "斜坡明挖工法的優點，下列何者正確？",
    "options": ["A. 適合無水乾燥地盤", "B. 機具進出方便", "C. 不需擋土設施費用", "D. 適合都市狹窄地區"],
    "answer": ["A", "B", "C"],
    "explanation": "斜坡明挖優點：適合乾燥地盤、機具方便、不需擋土設施。D錯誤：需要較大空間，不適合都市狹窄地區。"
  },
  {
    "question": "關於斜坡明挖工法，下列敘述何者有誤？",
    "options": ["A. 適合狹窄都市用地", "B. 無需任何排水措施", "C. 斜坡角度越陡越安全", "D. 適合所有地盤條件"],
    "answer": ["A", "B", "C", "D"],
    "explanation": "四項全錯：需要較大空間不適合狹窄都市；需排水措施；斜坡越陡越危險；地盤軟弱時不適合。"
  }
]
```

課本內若有【基本實務題】或【進階應用題】方框，判斷該題對應哪個知識點：

```js
"isExamFocus": true,
"examYears": ["103年土木高考", "103年鐵路員級"],
"examHint": "常考支撐構材名稱與功能、施工注意事項"
```

- `examYears`：從題目方框中的年份資訊提取
- `examHint`：簡短說明這個知識點在考試中常考的方向（一句話）

---

## 輸出檔案

- 檔名：`ch{章號}.js`，如 `ch3.js`、`ch4.js`
- 若 PDF 為 part 1，先輸出現有內容，等 part 2 丟進來後更新合併
- 存入 `/mnt/user-data/outputs/` 並用 `present_files` 提供下載

---

## 完成後告知使用者

輸出完成後，簡短說明：
1. 共幾張卡片
2. 各 section 的卡片數量
3. 標記了哪些考點（若有）
4. 若為 part 1，提示等待 part 2

使用者將 `.js` 檔存入 VS Code 專案的 `/data/` 資料夾後，網站即可自動讀取顯示。
